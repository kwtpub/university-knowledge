# 📚 Режим обучения

## Описание

Режим обучения позволяет систематически изучать билеты по темам и отслеживать свой прогресс в отдельной базе данных.

## Особенности

### 🎯 Отдельная база данных
- Все результаты обучения сохраняются в отдельном файле `src/data/learning-progress.json`
- Не влияет на основную базу билетов и историю ответов
- Хранит информацию о каждой попытке изучения билета

### 📊 Автоматическая отрисовка прогресса
После каждой сессии обучения автоматически создаётся Markdown файл с детальным прогрессом:
- `Прогресс_обучения_[Название_темы].md` в корне проекта
- Готов для использования в Obsidian
- Содержит статистику и детали по каждому билету

### 📈 Что отслеживается
- Количество попыток изучения каждого билета
- Текущий уровень понимания (ХОРОШО/СРЕДНЕ/ПЛОХО/НИКАК)
- Дата последней попытки
- Статистика по теме в процентах

## Как использовать

1. Запустите приложение: `npm start`
2. Выберите "Режим обучения по теме"
3. Выберите интересующую тему
4. Отвечайте на билеты, оценивая своё понимание
5. После завершения сессии:
   - Прогресс сохранится в базе обучения
   - Создастся/обновится Markdown файл с прогрессом

## Структура файла прогресса

```markdown
# 📚 Прогресс обучения: [Название темы]

## 📊 Статистика
- Всего билетов в обучении: X
- ХОРОШО (✅): X (XX.X%)
- СРЕДНЕ (⚠️): X (XX.X%)
- ПЛОХО (❌): X (XX.X%)
- НИКАК (❓): X (XX.X%)

---

## ✅ ХОРОШО (X)
### Билет №X
**Текст:** ...
- **Попыток:** X
- **Последняя попытка:** дата и время

## ⚠️ СРЕДНЕ (X)
...

## ❌ ПЛОХО (X)
...

## ❓ НИКАК (X)
...
```

## Интеграция с Obsidian

Файлы прогресса находятся в корне проекта и имеют расширение `.md`, что делает их совместимыми с Obsidian:

1. Откройте папку проекта в Obsidian как vault
2. Файлы с префиксом `Прогресс_обучения_` будут доступны для просмотра
3. Используйте внутренние ссылки Obsidian для навигации между темами
4. Можно создать дашборд со ссылками на все файлы прогресса
