eps должен быть ПОЛОЖИТЕЛЬНЫМ, чтобы:
  ✓ Расширить условие завершения цикла ВПЕРЕД (после B)
  ✓ Гарантировать включение точки B
  ✓ Компенсировать ошибки округления, которые могут сделать x > B

eps = h/2 — оптимальное значение:
  ✓ Достаточно большое, чтобы покрыть ошибки округления
  ✓ Достаточно маленькое, чтобы не добавить лишних итераций

Проблемы константного eps:

1. ❌ СЛИШКОМ БОЛЬШОЙ eps (например, 50):
   • Цикл выполняет множество лишних итераций
   • Вычисляются значения функции ВНЕ интервала [A, B]
   • Тратится лишнее время и ресурсы
   • Получаются ненужные данные

2. ❌ СЛИШКОМ МАЛЕНЬКИЙ eps (например, 0.001):
   • Может не хватить для больших шагов h
   • Точка B может быть пропущена при накоплении ошибок округления
   • Особенно критично для больших интервалов [A, B]

3. ❌ НЕ АДАПТИРУЕТСЯ к шагу h:
   • Один и тот же eps не подходит для разных интервалов
   • Нужно подбирать вручную для каждого случая
   • Не универсальное решение

Почему eps = h/2 оптимален:

✓ АВТОМАТИЧЕСКИ адаптируется к любому шагу h
✓ ПРОПОРЦИОНАЛЕН размеру шага (всегда половина)
✓ ГАРАНТИРУЕТ включение точки B для любых A, B, N
✓ УНИВЕРСАЛЬНОЕ решение для всех случаев
✓ МИНИМАЛЬНО необходимое значение (не добавляет лишних итераций)
