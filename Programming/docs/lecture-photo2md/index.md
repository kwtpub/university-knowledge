# lecture-photo2md

Методика и набор навыков для агента, который читает фото рукописных лекций и собирает структурированный Markdown-конспект.

## Цель агента

Агент получает папку с фото страниц лекции (`lect-ref/lect-N`) и формирует один файл конспекта (`lect/lect-N.md`) с:

- темой и датой лекции;
- ключевыми определениями;
- примерами кода/формул;
- выводами по разделам.

## Набор навыков агента

1. Навигация по проекту и инвентаризация входных файлов.
2. Чтение изображений страниц вручную (`view_image`) с распознаванием рукописного текста.
3. Восстановление структуры лекции по контексту (заголовки, переходы, примеры, выводы).
4. Доменно-ориентированная интерпретация (например, C/C++: `*`, `&`, `'\0'`, указатели, сигнатуры функций).
5. Работа с плохо читаемыми местами через увеличения/кропы (`magick`).
6. Осторожное нормализующее переписывание в чистый Markdown без потери смысла.
7. Валидация результата и фиксация файла в нужном пути.
8. Устойчивость к прерываниям: продолжение с текущего шага без потери прогресса.

## Инструменты и зачем они нужны

- `rg --files`, `ls -la`: быстро найти и проверить входные файлы.
- `view_image`: основной способ чтения страниц.
- `magick`: увеличить, обрезать и улучшить контраст для спорных участков.
- `sed -n`, `cat`: проверка и финальный просмотр сгенерированного `.md`.
- `apply_patch`: аккуратная запись итогового документа.

Примечание: OCR (`tesseract`) можно пробовать как вспомогательный канал, но для рукописных фото он часто шумный. Базовый процесс должен работать и без OCR.

## Пошаговый пайплайн (эталон)

1. Проверить вход и выход.
- Убедиться, что есть `lect-ref/lect-N/*.jpeg`.
- Проверить, существует ли `lect/lect-N.md`.

2. Определить порядок страниц.
- Сортировать по имени файла/времени съёмки.
- Читать последовательно, не перескакивая.

3. Считать страницы в черновик.
- Пройти все фото через `view_image`.
- Выписать: заголовки, определения, код, формулы, подписи к рисункам.

4. Обработать нечитабельные места.
- Сделать `magick ... -resize ... -sharpen ...`.
- При необходимости сделать кроп верх/середина/низ страницы.
- Повторно прочитать только спорные строки.

5. Собрать структуру лекции.
- Сначала скелет разделов.
- Затем заполнить каждый раздел фактами из фото.
- Кодовые фрагменты вынести в fenced-блоки.

6. Нормализовать текст.
- Исправлять только очевидные орфографические и форматные дефекты.
- Термины/сигнатуры/формулы сохранять максимально близко к записи.
- Если слово неразборчиво, восстановить по контексту и не добавлять выдуманные детали.

7. Записать в `lect/lect-N.md`.
- Единый формат: заголовок, секции, списки, код.
- Без лишней воды и повторов.

8. Проверить результат.
- `sed -n '1,260p' lect/lect-N.md`.
- Убедиться, что все страницы покрыты и нет пустых секций.

## Правила качества

1. Не выдумывать содержимое, которого нет на фото.
2. Если строка сомнительна, восстанавливать по ближайшему контексту (пример, тип данных, соседние строки).
3. Для кода приоритет: семантика и синтаксис языка, а не буквальное копирование каждого штриха.
4. В итоговом файле должен читаться учебный смысл, а не только «переписанный текст».
5. Сохранять одинаковый стиль между лекциями, чтобы материалы были сравнимыми.

## Шаблон структуры выходного файла

```md
# Лекция N: <тема> (<дата>)

## 1. <раздел>
Краткое содержание.

## 2. <раздел>
- Тезис
- Тезис

## 3. Пример
```cpp
// код из лекции
```

## 4. Выводы
- Что важно запомнить
```

## Поведение агента при долгих задачах

1. Давать короткие промежуточные апдейты по прогрессу.
2. При прерывании пользователем не начинать заново, а продолжать с последнего подтверждённого шага.
3. Отдельно фиксировать, что уже прочитано, а что осталось.

## Мини-чеклист перед сдачей

1. Все фото просмотрены.
2. Основные примеры и определения попали в `.md`.
3. Нет пустых или случайно обрезанных блоков.
4. Путь и имя файла строго соответствуют запросу пользователя.
