# Lab Solver Knowledge Base

## Назначение
Этот файл хранит практический контекст по оформлению и выполнению лабораторных работ в этом репозитории, чтобы повторно использовать единый подход.

## Рабочий контекст по текущим лабам
1. Основной язык: C++ (консольная программа).
2. Часто требуется чтение исходных данных из файла.
3. Для каждого набора данных использовать отдельный файл (`tests/inputN.txt`).
4. Вывод в текущих работах делается в терминал (`std::cout`), если в задании явно не требуется файл результата.
5. Обмен с подпрограммами должен идти через параметры, без глобальных переменных.

## Ожидания по стилю кода (по референсу)
1. В начале файла — верхняя шапка-комментарий (паспорт файла).
2. Разделы кода отделять крупными блочными комментариями.
3. Переменные поднимать наверх функций/блоков (объявление в начале).
4. Добавлять поясняющие комментарии к ключевым шагам (`открытие файла`, `проверки`, `чтение`, `вывод`).

## Шаблон проверки файла ввода
Использовать последовательность в стиле референса:
1. `fin.open(...)`.
2. `fin.fail()` -> файл не найден, выход с кодом ошибки.
3. `fin.peek() == EOF` -> файл пустой, выход с кодом ошибки.
4. Проверка `fail()` после каждого чтения.
5. Для ошибок чтения строк/массивов различать `eof()` и форматную ошибку.
6. Явно закрывать файл `fin.close()`.

## Шаблон отчета
Отчет хранить в `lab-X/Отчет.md` и включать разделы:
1. `Задание`.
2. `Структурная схема алгоритма (блок-схема)` (Mermaid).
3. `Псевдокод (по ГОСТу)`.
4. `Код (код программы)`.
5. `Результаты работы программы` (несколько тестов с входом/выводом).
6. `Вывод`.

Рекомендуемый текст вывода:
- Разработка завершена на том основании, что:
- 1. Ожидаемые результаты совпали с полученными.
- 2. Считаем набор тестов полным.

## Чеклист перед сдачей
1. Программа компилируется без ошибок.
2. Программа проходит тесты на разные случаи (нормальный, крайний, без совпадений, равенство условий и т.д.).
3. Оформление кода соответствует референсу (комментарии + переменные наверху).
4. Отчет заполнен по шаблону.
5. В `task-input.md` зафиксированы постановка, формат входа/выхода, идея решения.
