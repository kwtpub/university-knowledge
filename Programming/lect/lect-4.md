# Лекция 4: Массивы как параметры функций (05.03.2026)

## 1. Связь со способами передачи параметров

В начале лекции повторены варианты передачи параметров:

1. по значению;
2. по адресу:
- по указателю;
- по ссылке;
3. передача по адресу с `const` (без изменения данных).

Переход к теме: как передавать массивы в функции.

## 2. Что реально передается при вызове с массивом

Если в функцию передается массив, фактически передается адрес его первого элемента.

Идея из конспекта:

```cpp
int main() {
    int AVec[50];
    Convert(AVec);
}
```

Эквивалентно по смыслу параметра:

```cpp
int Convert(int *xPtr);
// и
int Convert(int xVec[]);
```

Для параметров функции записи `int *xPtr` и `int xVec[]` в этом контексте эквивалентны.

## 3. Последствия передачи массива по адресу

1. Копирования всего массива нет, вызов эффективнее.
2. Функция работает с исходными данными и может их изменить.
3. Размер массива «теряется» в параметре, поэтому длину нужно передавать отдельным аргументом.

Из-за этого типичный интерфейс:

```cpp
double Func(int *xVec, int xSizeVec);
// или
double Func(int xVec[], int xSizeVec);
```

## 4. Пример `sumup(...)`

Разобран базовый пример суммирования через указатель:

```cpp
int sumup(int *xPVec, int xLenVec) {
    int sum = 0;
    for (int i = 0; i < xLenVec; i++) {
        sum = sum + *(xPVec + i);
        // эквивалентно: sum += xPVec[i];
    }
    return sum;
}
```

Вызов:

```cpp
const int MMAX = 80;
int main() {
    int AVec[MMAX];
    int LenVec = 20;
    int s = sumup(AVec, LenVec);
}
```

## 5. Пример `Ex04_05.cpp`: три формы доступа

В конспекте показаны три функции среднего значения массива:

```cpp
double Average1(int xVec[], int xSizeVec);
double Average2(int *xVec, int xSizeVec);
double Average3(int &zVec, int xSizeVec); // ссылка на 1-й элемент
```

### Average1: параметр как `xVec[]`

- внутри функции `xVec` уже ведет себя как указатель;
- выражение `sizeof(xVec) / sizeof(int)` не дает реальный размер исходного массива;
- поэтому длину надо брать из `xSizeVec`.

Также отдельно показано, что для глобального массива `globVec` размер можно вычислить по месту объявления:

```cpp
sizeof(globVec) / sizeof(int)
```

### Average2: параметр как `int*`

Суммирование через арифметику указателей:

```cpp
sum += *(xPVec + i);
```

В тетради это сравнивается с индексной формой и печатаются адреса/значения для демонстрации эквивалентности.

### Average3: ссылка на первый элемент

Идея: если передать ссылку на нулевой элемент, можно идти по памяти от `&zVec`:

```cpp
sum += *(&zVec + i);
```

Комментарии из конспекта:

- `zVec` — значение нулевого элемента;
- `&zVec` — адрес нулевого элемента;
- `*(&zVec + i)` — значение `i`-го элемента.

Вызов в `main` делается через первый элемент:

```cpp
Average3(AVec[0], MMAX);
```

## 6. Важные выводы лекции

1. Передача массива в функцию по параметру `xVec[]` не копирует массив, а передает адрес.
2. Формы `int xVec[]` и `int *xVec` для параметров функции эквивалентны.
3. Без отдельного параметра длины корректно обойти массив нельзя.
4. Любая функция, принимающая массив по адресу/указателю, потенциально может изменить исходный массив.
5. Для безопасного «только чтения» в таких функциях нужно использовать `const`.
