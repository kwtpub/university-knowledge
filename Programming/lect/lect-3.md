# Лекция 3: Передача параметров и указатели на массивы (26.02.2026)

## 1. Передача параметров в функции

Продолжение темы с прошлой лекции:

- при передаче по значению функция работает с копиями аргументов;
- при передаче по адресу функция получает доступ к исходным данным;
- исходные значения аргументов могут быть модифицированы.

В конспекте выделены 2 формы передачи по адресу:

1. по указателю;
2. по ссылке.

## 2. Ключевая идея про ссылку

Ссылка описана как специальный способ доступа к объекту по его адресу.
Практический смысл: внутри функции можно работать с исходной переменной без явного копирования.

## 3. Схема примера `Ex03_05.cpp`

В тетради разобран пример с глобальной переменной и 4 вариантами функций:

```cpp
int globVar = 4;

void f1_V1(int x, int y);                 // передача по значению
void f2_Ref(int &x, int &y);              // передача по ссылке
void f3_Pt2(int *x, int *y);              // передача по указателю
void f4_Ptr_No(const int *x, const int *y); // по указателю без права изменения
```

Идея примера:

- `f1_V1` меняет только локальные копии `x`, `y`, но меняет `globVar`;
- `f2_Ref` меняет исходные `a`, `b` через ссылки;
- `f3_Pt2` меняет исходные `a`, `b` через указатели;
- `f4_Ptr_No` демонстрирует запрет изменения данных через `const`.

## 4. Каркас `main()` из примера

```cpp
int main() {
    int a = 1, b = 2;

    // печать начальных значений
    // вызов f1_V1(a, b)
    // печать значений после f1
    // вызов f2_Ref(a, b)
    // печать значений после f2
    // вызов f3_Pt2(&a, &b)
    // печать значений после f3
}
```

По подписям в конспекте видно, что вывод делается по шагам (`main`, `f1`, `f2`, `f3`) для сравнения эффекта каждого способа передачи.

## 5. Реализация `f1_V1`: передача по значению

```cpp
void f1_V1(int x, int y) {
    x = x + 10;   // изменение копии
    y = y + 10;   // изменение копии
    globVar = globVar + 10; // глобальная переменная меняется
}
```

После возврата в `main`:

- `a` и `b` остаются прежними;
- `globVar` уже изменена.

## 6. Реализация `f2_Ref`: передача по ссылке

```cpp
void f2_Ref(int &x, int &y) {
    x = x + 10;   // изменение оригинала
    y = y + 15;   // изменение оригинала
    globVar += 20;
}
```

После возврата в `main` значения исходных переменных уже новые.

## 7. Реализация `f3_Pt2`: передача по указателю

```cpp
void f3_Pt2(int *x, int *y) {
    *x = *x + 10; // изменение оригинала
    *y = *y + 15; // изменение оригинала
    globVar += 20;
}
```

Вывод из конспекта:
если функция должна менять переданные данные, можно использовать передачу по адресу (указатели/ссылки).

## 8. Почему передача по адресу часто эффективнее

Для больших структур/объектов передача по адресу быстрее, чем передача целого объекта по значению (нет дорогостоящего копирования).

Риск: по адресу можно случайно изменить исходные данные.

Решение из лекции: использовать `const` у параметров, если изменение не требуется.

## 9. Передача по адресу без права изменения

Пример из тетради:

```cpp
void f4_Ptr_No(const int *x, const int *y) {
    *x = *x + 10; // error
    *y = *y + 15; // error
}
```

`const` защищает данные от модификации внутри функции и помогает компилятору ловить ошибки.

## 10. Указатели на массивы

Основные тезисы:

- имя массива связано с адресом первого элемента;
- `arr` и `&arr[0]` указывают на начало массива;
- массивы и указатели тесно связаны, но это не одно и то же;
- имя массива нельзя переназначить.

Пример:

```cpp
int AVec[3] = {1, 2, 3};
int BVec[3];

BVec = AVec; // нельзя
```

## 11. Пример `Ex04_03.cpp`: указательный обход массива

```cpp
int iArr[4], *pInt;
double dArr[4], *pDb;

pInt = iArr;
pDb = &dArr[0];

for (int i = 0; i < 4; i++) {
    *(pInt + i) = -(2 * i + 11);
    dArr[i] = (2 * i + 1) / 2.0;
    // печать адресов и значений через указатели
}
```

Эквивалентные записи доступа:

- `pInt + i` и `&iArr[i]`;
- `*(pInt + i)` и `iArr[i]`;
- `iArr[2]`, `*(pInt + 2)`, `*(2 + pInt)`, `*(iArr + 2)`, `2[iArr]`.

Важно не путать:

- `*iArr + 2` (арифметика со значением первого элемента);
- `*(iArr + 2)` (доступ к элементу с индексом `2`).
