# Лекция 2: Указатели (19.02.2026)

Из конца прошлой лекции в конспекте отдельно подчеркнуто:
`Глобальная переменная — враг народа!`

## 1. Зачем нужны адреса

При хранении данных программа должна понимать:

1. где в памяти лежит значение (адрес);
2. какой это тип данных (чтобы знать, как интерпретировать биты и какие операции допустимы).

Для работы с адресами используется специальная форма записи: указатели.

## 2. Оператор взятия адреса

- `&N` — адрес переменной `N`;
- `N` — само значение.

Оператор `&` возвращает адрес переменной.

## 3. Пример взятия адресов (Ex03-01.cpp)

```cpp
int main() {
    char ch1 = 'W';
    char ch2 = 'R';
    int iVar = 1235;
    long long lVar = 65535;
    double dVar = 2718281828459045; // в тетради число записано слитно

    cout << "ch1 = " << ch1
         << "\tAddress of ch1 = " << &ch1
         << " " << (int)&ch1 << endl;
    // Аналогично: ch2, iVar, lVar, dVar
}
```

## 4. Указатели и разыменование

В конспекте выделено:

- `int *pAge;` — указатель на `int`;
- `int Age;` — обычная переменная типа `int`.

Смысл `*` зависит от контекста:

- в объявлении (`int *p`) — это признак указателя;
- в выражении (`*p`) — это разыменование (доступ к значению по адресу).

Пример:

```cpp
int N = 20;
int *pAge;
pAge = &N;
cout << *pAge; // 20
```

## 5. Пример с `myAge` (Ex03-02.cpp)

```cpp
int main() {
    int myAge;
    int *pAge = 0;

    myAge = 5;
    cout << "myAge = " << myAge << endl;

    pAge = &myAge;
    cout << "pAge = " << pAge << "\t*pAge = " << *pAge << endl;

    *pAge = 7;
    cout << "myAge = " << myAge << endl;
    cout << "*pAge = " << *pAge << endl;

    myAge = 20;
    cout << myAge << " " << *pAge << endl;

    cout << "sizeof(pAge) = " << sizeof(pAge) << endl;
    cout << "&myAge = " << &myAge << endl;
    cout << "pAge = " << pAge << "\t&pAge = " << &pAge << endl;
}
```

Вывод из примера:

- `myAge` и `*pAge` дают доступ к одному и тому же значению;
- `pAge` хранит адрес `myAge`;
- `&pAge` — адрес самой переменной-указателя.

## 6. Типизация указателей

Указатель всегда типизирован: он указывает на конкретный тип данных.

Тип указателя нужен компилятору, чтобы:

- корректно читать значение по адресу;
- правильно выполнять арифметику указателей.

Пример объявления:

```cpp
int *p1, *p2; // два указателя
int *p1, p2;  // p1 — указатель, p2 — обычный int
```

Указатель со значением `0` в тетради назван нулевым указателем.

## 7. Арифметика указателей

Для арифметики используется размер типа, на который указывает указатель.

В конспекте отмечено:

- при `p++` адрес сдвигается не на 1 байт, а на `sizeof(T)`;
- для `double*` шаг равен 8 байт.

Пример (Ex03-04.cpp):

```cpp
int main() {
    double dNumber;
    double *dPtr = &dNumber;
    double *dPtr0;
    dPtr0 = dPtr;

    cout << "dPtr = " << dPtr << " " << (int)dPtr << endl;
    dPtr++;
    cout << "after ++dPtr = " << dPtr << " " << (int)dPtr << endl;
    cout << "count of elems double = " << dPtr - dPtr0 << endl;
}
```

## 8. Передача параметров в функцию

В тетради выделены способы:

1. по значению;
2. по адресу:
   - по указателю;
   - по ссылке.

Комментарий из лекции: при передаче по значению создается временная копия объекта; это может быть дорого для больших данных.
